Overview

What is a Custom Agent?

A custom agent is an AI application purpose-built for your workflow — not a generic chatbot or a pre-packaged product, but a tailored agent system that speaks your domain language, connects to your tools, and executes the specific logic your business needs.

Unlike off-the-shelf AI products, a custom agent is built from the ground up to solve one thing well. It may be a voice agent that handles customer calls in Vietnamese, a coding assistant that knows your codebase's conventions, a RAG pipeline over your proprietary documents, or a multi-agent system that orchestrates research, drafting, and review.

Purpose-built

Designed for your specific domain, workflow, and data — not a generic product

Developer-controlled

Full access to every layer — model, memory, tools, security, deployment

Production-hardened

Rust-native performance, WASM sandboxing, and zero-cost abstractions

Define agent workflows as directed graphs with conditional branching, parallel execution, sub-agent delegation, and human-in-the-loop checkpoints. Compile and validate at build time.

SQLite-backed storage with vector embeddings for semantic memory. Sessions persist across invocations, and agents maintain context, preferences, and learned patterns over time.

Streaming & Realtime

First-class support for streaming responses, server-sent events, and WebSocket connections. Build realtime agent applications that stream tokens, audio, and tool outputs.

Security

Built-In Security Model

ADK-Rust is built in Rust, which eliminates entire classes of memory-safety vulnerabilities at compile time. On top of that, its WASM sandbox provides a second layer of defence for tool execution.

  • • WASM dual-metered sandbox — tool code cannot escape its environment
  • • Fuel metering — prevent runaway execution and denial-of-service
  • • Workspace-confined file operations — agents cannot access files outside their scope
  • • Env-cleared subprocesses — no leaked environment variables or secrets
  • • Rust's memory safety — no buffer overflows, use-after-free, or null pointer dereferences
  • • Secret zeroization — API keys and credentials wiped from memory when not in use

Considerations

What to Know Before You Start

  • A development project, not a product. Building a custom agent requires a software development team. You are not configuring a pre-built tool — you are building an application from scratch.
  • Rust expertise helps. While we handle the heavy lifting, teams that understand Rust will get the most value from customising and extending their agent over time.
  • Ongoing operating cost. The framework is free (Apache 2.0). You pay for LLM inference via your chosen provider and infrastructure costs for hosting your agent binary.
  • Iterate toward production. Start with a focused pilot, validate with real users, then expand scope. Custom agents are most successful when built incrementally.

FAQ

Common Questions

How is a custom agent different from a Business Agent?
A Business Agent is a pre-built product (OpenFang) that we deploy and configure for you — it is an always-on agent OS that manages communications and automations. A Custom Agent is built from scratch with ADK-Rust to solve a specific problem. Think of it as the difference between buying a turnkey piece of software and commissioning custom software development.
Is this just a chatbot?
No. While a custom agent can include chat interfaces, it is fundamentally an application that uses AI to execute specific logic — voice processing, document analysis, multi-step research, tool orchestration, and more. Chat is one interface among many.
Do I need Rust expertise?
Not necessarily. We handle the ADK-Rust development. However, if your team plans to maintain and extend the agent after launch, some Rust familiarity is beneficial. We can include knowledge transfer and training as part of the engagement.
What infrastructure do I need?
ADK-Rust compiles to a single binary that runs on any Linux or macOS system. Deployment options include a VPS, dedicated server, Docker container, or edge environment. We recommend based on your latency, scale, and compliance requirements.
Can I use my own LLM provider?
Absolutely. ADK-Rust supports 26+ providers including Anthropic, Gemini, OpenAI, Groq, DeepSeek, and any OpenAI-compatible endpoint. You choose the model per agent or per task — no code changes needed to switch.
