{# ─── What Is Dioxus ─────────────────────────── #}

Overview

What is Dioxus?

Dioxus is a Rust framework for building fullstack web, desktop, and mobile applications with a single codebase. It provides a React-like component model with signals-based reactivity, server functions for backend logic, and a built-in bundler that deploys to every major platform.

Unlike Electron or React Native, Dioxus apps are compiled to native code with zero virtual machine overhead. On the web, Dioxus renders directly to the DOM via WebAssembly with bundles comparable to React (~50kb hello world). On desktop, it renders through the platform's native WebView or experimentally with WGPU/Skia, producing portable binaries under 3mb. On mobile, Dioxus targets iOS and Android with direct JNI and Objective-C calls — no JavaScript bridge required.

{# ─── Cross-Platform Architecture ─────────────── #}

Architecture

Cross-Platform by Design

Write your app once with the same Rust components, state management, and business logic. Then compile and bundle for each target with a single CLI command.

🌐

Web

Render directly to the DOM via WebAssembly. Pre-render with Server-Side Rendering and hydrate on the client. A simple "hello world" comes in around 50kb — comparable to React. Built-in dev server with hot reloading for rapid iteration.

  • • WASM compilation with code splitting
  • • SSR, hydration, and HTML streaming
  • • Fullstack server functions with Axum
  • • .avif generation, .wasm compression
💻

Desktop

Render using the platform's native WebView — or experimentally with WGPU (Freya) or Skia. Zero-config setup: dx serve --platform desktop builds for macOS, Linux, and Windows. Portable binaries under 3mb with full native system access — no IPC overhead.

  • • Native WebView, WGPU, or Skia rendering
  • • Full filesystem, network, and system APIs
  • • Menubar, tray icon, global shortcuts
  • • <3mb portable binaries
📱

Mobile

Build native .ipa and .apk files for iOS and Android. Call directly into Java (JNI) and Objective-C with minimal overhead — no JavaScript bridge. From "hello world" to running on device in seconds with dx serve --platform android.

  • • iOS and Android native builds
  • • Direct JNI and ObjC FFI
  • • iOS widgets and live activities
  • • Kotlin, Java, and Swift FFI interfaces
{# ─── Fullstack Capabilities ──────────────────── #}

Features

Everything You Need to Ship

Server Functions

Add backend logic directly alongside your frontend components. Server functions compile to Axum routes for the server and auto-generated async clients for the browser — with WebSocket support, streaming, and error handling.

Signals-Based Reactivity

Fine-grained reactive state management inspired by SolidJS and Svelte. No virtual DOM, no diffing — components update precisely where the data changes. Combines the best of React, Solid, and Svelte.

Type-Safe Routing

First-party router with type-safe routes, nested layouts, link generation, and query parameter handling. Routes are validated at compile time — no more broken links or incorrect params.

Subsecond Hot-Reloading

Dioxus's Subsecond engine hot-patches Rust code changes in under a second across all platforms — web (WASM), desktop, and mobile. Markup, assets, and Rust code all hot-reload without restarting your app.

Radix Primitives

First-party implementation of 28 unstyled, accessible Radix primitives built directly for Dioxus. Keyboard shortcuts, ARIA support, and full cross-platform compatibility — mix, match, and restyle freely.

Asset Bundling

Integrated bundler for web, desktop, and mobile. Automatic .avif generation, .wasm compression, CSS minification, and hash-based asset cache invalidation. dx bundle packages everything.

FAQ

Common Questions

Why Dioxus over Flutter, React Native, or Tauri?
Dioxus offers the best of all three: the component model and developer experience of React, the native performance of Rust (no VM, no garbage collector), and cross-platform reach without JavaScript bridges. Unlike Flutter, Dioxus renders with standard HTML/CSS. Unlike React Native, there is no JavaScript layer — your code compiles directly to native. Unlike Tauri, Dioxus includes server functions, SSR, and mobile support out of the box — it is a fullstack framework, not just a desktop shell.
Do I need to know Rust?
Not necessarily. We handle the Dioxus development. However, if your team plans to maintain and extend the application after launch, Rust familiarity is beneficial. We can include knowledge transfer and training as part of the engagement.
Can I deploy to iOS and Android app stores?
Yes. Dioxus builds native .ipa and .apk files for iOS and Android respectively. The dx bundle command handles packaging with the correct entitlements, splash screens, and URL schemes. You can customise every aspect of iOS and Android builds via Dioxus.toml — permissions, AndroidManifest.xml, Info.plist, and more.
How does Dioxus handle state management?
Dioxus uses signals-based reactivity — fine-grained reactive primitives inspired by SolidJS and Svelte. use_signal creates reactive state, use_memo derives computed values, and use_effect handles side effects. There is no virtual DOM: components re-render precisely where the signal changes, giving excellent performance by default.
Can I use my existing Rust crates and code?
Absolutely. Dioxus is a Rust framework — you have access to the entire Rust ecosystem. Use any crate for database access (SQLx, Diesel), serialization (Serde), HTTP clients (reqwest), async runtimes (Tokio), ML inference (Candle, Burn), or any other Rust library. Your business logic, data models, and algorithms all live in the same language as your UI.
What infrastructure do I need?
For web deployment, Dioxus compiles to static files (WASM + HTML + CSS) that can be served from any CDN, cloud storage bucket, or web server. For fullstack apps with server functions, you need a host that runs Rust binaries — a VPS, Docker container, or serverless environment. Desktop and mobile apps bundle everything into platform-native installers. We help you choose the right deployment model based on your requirements.
