Overview

What is a Business Agent?

A business agent is a persistent AI assistant that lives on your infrastructure — not in a company's cloud. It plans, acts, observes results, and loops until tasks are done. Unlike chatbots or copilots, it maintains state, memory, and learned skills across sessions, so it doesn't start from zero every time you talk to it.

Think of it as a capable team member who works around the clock: triaging emails, checking calendars, running reports, syncing data across tools, and alerting you when something needs attention. It reaches you on whatever messaging platform you already use and gets more useful the longer it runs.

{# ─── Security & Considerations ──────────────── #}

Security

Minimising the Blast Radius

The more capabilities you give an agent, the more useful — and more powerful — it becomes. The same access that enables email triage (read, write, delete) is also a potential attack surface. Our approach is to embrace the power while ruthlessly minimising the blast radius.

  • • Dedicated hardware or VLAN isolation for the agent
  • • MCP gateway pattern — credentials never live on the agent machine
  • • Least-privilege tool permissions (read-only where possible)
  • • Firewall rules limiting agent-to-NAS communication to specific ports
  • • Off-site backups for data preservation
  • • Permission prompts for destructive or sensitive actions

Considerations

What to Know Before You Start

  • Not a plug-and-play product. Setting up a business agent securely requires understanding of servers, networking, OAuth, and permission models.
  • Ongoing operating cost. The software is free (MIT). You pay for the LLM inference via your chosen provider — typically $20-200/month depending on usage and model choice.
  • Model selection matters. Start with frontier models (GPT-5, Claude 4) to define workflows, then swap to smaller, cheaper models once processes are locked in.
  • You are the operator. A business agent needs oversight, especially in the early days. Review its work, tune its permissions, and iterate on its skills.

{# ─── FAQ ─────────────────────────────────── #}

FAQ

Common Questions

How is a business agent different from ChatGPT or Claude?
Standard chatbots are stateless — each conversation starts fresh. A business agent runs persistently, remembers everything across sessions, writes its own skills, executes scheduled tasks, and takes multi-step actions on your behalf. It is an employee, not a conversation.
Is my data safe?
Yes. Hermes Agent is self-hosted on your infrastructure. All data stays in ~/.hermes/ on your machine. Zero telemetry, zero data collection, MIT license — you can audit every line of code. We implement additional security layers including VLAN isolation and MCP gateways to minimise blast radius.
How much does it cost to run?
The software is free (MIT license). You pay for the LLM inference — typically $20-200/month depending on the model and usage volume. A $5/month VPS is enough to run the agent continuously. We help you select the right model and infrastructure for your budget.
Do I need technical skills to use it?
Yes — setting up and maintaining a business agent securely requires comfort with servers, networking, OAuth, and permission models. That is exactly where we come in. We handle the infrastructure and security so you can focus on using the agent, not configuring it.
What messaging platforms are supported?
Discord, Telegram, Slack, WhatsApp, Signal, and direct CLI. All from a single gateway process. Voice memo transcription is also supported on compatible platforms.
Can I use my own AI model?
Absolutely. Hermes is model-agnostic and connects to 200+ models via OpenRouter, OpenAI, Anthropic, Nous Portal, local Ollama/vLLM, or any OpenAI-compatible endpoint. No code changes needed to switch models.
